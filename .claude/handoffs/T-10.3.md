# T-10.3: Enterprise SSO Integration - Handoff

## Summary
Implemented Enterprise SSO support supporting both SAML 2.0 and OIDC protocols. This allows enterprise tenants to bring their own Identity Provider (IdP) like Okta, Azure AD, or Google Workspace.

## Deliverables

### Core Logic (`crates/ramp-core/src/sso/`)
- `mod.rs`: `SsoService`, `SsoConfig`, `RoleMapping` definitions and role mapping logic.
- `oidc.rs`: `OidcProvider` implementation for Okta, Azure AD, Google, Auth0.
- `saml.rs`: `SamlProvider` implementation (SAML 2.0 AuthRequest generation and Response parsing).

### API Handlers (`crates/ramp-api/src/handlers/auth/sso.rs`)
- `GET /v1/auth/sso/:provider/login`: Initiates SSO flow (redirects to IdP).
- `GET /v1/auth/sso/:provider/callback`: Handles IdP callback (code/SAMLResponse exchange).

### Database (`migrations/018_enterprise_sso.sql`)
- `identity_providers` table: Stores IdP configuration (encrypted).
- `sso_sessions` table: Tracks active SSO sessions.
- `sso_provider_type` & `sso_protocol` enums.

## Configuration
- `SsoConfig` allows defining:
  - Protocol (OIDC/SAML)
  - Provider specific config (ClientId/Secret or EntityID/Cert)
  - Role Mappings (e.g., IdP Group "Admins" -> RampOS "tenant_admin")
  - Default Role (e.g., "viewer")

## Integration Notes
- `SsoService` added to `AppState`.
- Routes mounted at `/v1/auth/sso`.
- Needs to be initialized in `main.rs` with `SsoService::new()`.
- **Security**: SAML XML signature verification is currently a placeholder/simplified. For production, integrate with `libxml2` or a dedicated Rust SAML crate that supports XML-DSIG.

## Testing
- Unit tests in `mod.rs` for role mapping logic.
- Unit tests in `oidc.rs` for provider configuration factories.
- Manual testing required with mock IdP (e.g., Keycloak or Auth0 free tier).
