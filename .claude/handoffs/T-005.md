# Handoff: T-005 - Network Policies

## Summary
Implemented strict Kubernetes NetworkPolicies to enforce a "default deny" security posture. Only explicitly required traffic flows are allowed.

## Deliverables
- `k8s/base/network-policy.yaml`: Contains 5 NetworkPolicy resources.
- `k8s/NETWORKING.md`: Documentation of the network security architecture.
- `k8s/jobs/migration-job.yaml`: Updated to include `app: rampos-migration` label for policy matching.

## Network Security Details
1. **Default Deny**: `default-deny-all` policy blocks all traffic by default.
2. **API Isolation**:
   - Accepts traffic ONLY from Ingress Controller.
   - Can egress ONLY to DB, Redis, NATS, DNS, and External HTTPS (443).
3. **Database Isolation**:
   - Postgres/Redis accept traffic ONLY from the API server (and migration job for Postgres).
   - No internet access for data stores.
4. **NATS Clustering**:
   - Policies configured to allow peer-to-peer discovery and clustering.

## Verification
- Reviewed labels across `deployment.yaml`, `statefulset.yaml`, and `job.yaml` to ensure selectors match.
- Added DNS egress permissions for API and NATS (required for service discovery).
- Added external HTTPS egress for API (required for webhooks/3rd party APIs).

## Notes for Next Steps

- **Monitoring**: Current monitoring configuration (ServiceMonitor) has incorrect labels (`app: ramp-api` vs `app: rampos-server`). Network policies currently **BLOCK** monitoring traffic. This needs to be addressed in a future task (or T-006) by fixing the ServiceMonitor labels AND adding a `monitoring-policy` to allow ingress from the monitoring namespace.
