# Task T-4.3.1: E2E Payin Flow Test

## Summary
Implemented a full End-to-End test for the Payin flow using `testcontainers` for isolated PostgreSQL integration testing.

## Artifacts
- **Test File**: `crates/ramp-api/tests/e2e_payin_test.rs`
- **Dependencies**: Added `testcontainers`, `testcontainers-modules`, `reqwest` to `crates/ramp-api/Cargo.toml`

## Test Coverage
The test covers the following flow:
1. **Setup**:
   - Starts PostgreSQL container
   - Runs database migrations
   - Creates Tenant and User records
   - Configures Rails Adapter (VCB)
2. **Execution**:
   - **Step 1**: `POST /v1/intents/payin` - Creates intent
   - **Step 2**: `POST /v1/intents/payin/confirm` - Simulates Bank Webhook
3. **Verification**:
   - **API Response**: Verifies status transitions (`INSTRUCTION_ISSUED` -> `COMPLETED`)
   - **Database**: Verifies Intent state in DB
   - **Ledger**: Verifies Double-Entry Ledger records (Debit Bank, Credit User Liability)
   - **Balance**: Verifies User Balance update

## Environment Note
The test requires a Docker environment to run `testcontainers`. If running in CI/CD, ensure Docker is available.
