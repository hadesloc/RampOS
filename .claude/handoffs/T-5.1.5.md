# Handoff: T-5.1.5 Add Tier Management API

## Summary
Implemented the Admin API endpoints for tier management. This includes listing tiers, getting user tier info, upgrading/downgrading user tiers, and getting user limits.

## Changes
- Created `crates/ramp-api/src/handlers/admin/tier.rs`:
  - Defined DTOs: `TierConfiguration`, `UserTierInfo`, `TierChangeRequest`, `UserLimitInfo`
  - Implemented handlers: `list_tiers`, `get_user_tier`, `upgrade_user_tier`, `downgrade_user_tier`, `get_user_limits`
  - Added helper `check_admin_key` to validate `X-Admin-Key` header
- Updated `crates/ramp-api/src/handlers/admin/mod.rs`:
  - Exported `tier` module
- Updated `crates/ramp-api/src/router.rs`:
  - Registered tier management routes under `/v1/admin`

## Verification
- Added unit tests in `tier.rs` to verify `X-Admin-Key` validation logic.
- Checked that routes are correctly registered in the router.
- Verified DTO structures match the requirements.

## Notes
- Service calls are currently mocked with TODOs as the underlying `TierService` and `LimitService` were not available/in-scope.
- Endpoints are mounted at `/v1/admin/...` to match existing admin routes structure.
- `X-Admin-Key` is currently hardcoded to `"admin-secret-key"` as a placeholder. Should be moved to configuration in the future.
