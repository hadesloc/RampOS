# Handoff: T-3.1.5 Intent Timeout Handling

## Summary
Implemented background job to handle intent timeouts and state transitions.

## Changes
1. **Core Job**: `crates/ramp-core/src/jobs/intent_timeout.rs`
   - Scans for expired intents every minute
   - Transitions state:
     - `PAYIN_VND` -> `EXPIRED`
     - `PAYOUT_VND` -> `TIMEOUT`
   - Publishes `intent.status_changed` events

2. **Repository**: `crates/ramp-core/src/repository/intent.rs`
   - Added `list_expired` method to find intents where `expires_at < NOW()`
   - Robust filtering: Excludes `COMPLETED`, `EXPIRED`, `CANCELLED`, `TIMEOUT`, `REJECTED_BY_POLICY`, `BANK_REJECTED`, `SUSPECTED_FRAUD` to prevent loops and overwriting terminal states.

3. **Service Logic**: `crates/ramp-core/src/service/payout.rs`
   - Updated payout creation to set `expires_at` to 1 hour (default)
   - Payin creation already sets `expires_at` to 24 hours

4. **API Integration**: `crates/ramp-api/src/main.rs`
   - Registered and started `IntentTimeoutJob` in background

## Verification
- Unit tests in `intent_timeout.rs` cover expiry logic for Payin and Payout.
- `MockIntentRepository` supports `list_expired` (updated to match real filters).
- Integration points checked in `main.rs`.

## Next Steps
- Monitor job performance with telemetry (logs added).
- Consider adding configuration for timeout durations (currently hardcoded 24h/1h).
