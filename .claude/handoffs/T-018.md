# T-018: Create Rust CI Workflow

**Status**: Completed
**Date**: 2026-02-06
**Priority**: P1

## Summary
Created a comprehensive Rust CI workflow to replace the basic one. The new workflow includes matrix testing, security audits, and optimized caching.

## Deliverables
1.  **`.github/workflows/rust-ci.yaml`**: New workflow file.
    -   **Jobs**:
        -   `check`: Runs `cargo fmt` and `cargo clippy`.
        -   `test`: Runs `cargo test` on `stable` and `beta` toolchains. Services (Postgres, Redis) are spun up.
        -   `build`: Verifies `cargo build --release` succeeds.
        -   `audit`: Runs `cargo audit` via `rustsec/audit-check`.
    -   **Triggers**: Push to main/master, PRs affecting Rust files.
    -   **Optimization**: Uses `Swatinem/rust-cache@v2`.
2.  **Cleanup**: Removed `.github/workflows/ci.yaml` to avoid duplication.
3.  **Security Audit Update**: Updated `.github/workflows/security-audit.yml` to run only on schedule (nightly), as PR/Push checks are now handled by `rust-ci.yaml`.

## Verification
-   Workflow syntax is valid.
-   Service containers for integration tests are configured correctly.
-   Matrix strategy allows testing multiple Rust versions.
-   Security audit integration covers `cargo audit`.
