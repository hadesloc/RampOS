# T-7.9: Compliance Audit Trail - Handoff

## Status: COMPLETED

## Summary
Implemented an immutable compliance audit trail with hash chain for regulatory inspections. This provides a tamper-evident log of all compliance-related events that can be exported for regulators.

## API Endpoints

### GET /v1/admin/audit/compliance
List compliance audit entries with filtering.

**Query Parameters:**
- `limit` (default: 50)
- `offset` (default: 0)
- `eventType` - Filter by event type
- `actorId` - Filter by actor
- `resourceType` - Filter by resource type
- `resourceId` - Filter by resource ID
- `fromDate` - Start date (ISO 8601)
- `toDate` - End date (ISO 8601)

**Response:**
```json
{
  "data": [...],
  "total": 100,
  "limit": 50,
  "offset": 0
}
```

### GET /v1/admin/audit/verify
Verify the integrity of the audit chain.

**Response:**
```json
{
  "isValid": true,
  "totalEntries": 1000,
  "verifiedEntries": 1000,
  "firstInvalidSequence": null,
  "errorMessage": null,
  "verifiedAt": "2026-02-06T12:00:00Z"
}
```

### GET /v1/admin/audit/export
Export audit log for regulators.

**Query Parameters:**
- `fromDate` - Start date (ISO 8601)
- `toDate` - End date (ISO 8601)
- `format` - "json" (default) or "csv"

**Response (JSON):**
```json
{
  "tenantId": "...",
  "exportedAt": "...",
  "totalEntries": 100,
  "chainVerified": true,
  "entries": [...]
}
```

**Response (CSV):**
Returns downloadable CSV file.

## Event Types Logged
- `compliance_decision` - General compliance decisions
- `document_submitted` - Compliance document submissions
- `rule_changed` - Rule modifications
- `user_action` - User-initiated actions
- `kyc_tier_change` - KYC tier upgrades/downgrades
- `transaction_approval` - Transaction approvals
- `transaction_rejection` - Transaction rejections
- `aml_rule_modification` - AML rule changes
- `sar_submission` - SAR filings
- `ctr_submission` - CTR filings
- `license_status_change` - License status changes
- `sanctions_check` - Sanctions screening results
- `pep_check` - PEP screening results

## Security Features
1. **Append-only** - Triggers prevent UPDATE/DELETE
2. **Hash chain** - Each entry links to previous via SHA256
3. **RLS** - Tenant isolation at database level
4. **Chain verification** - API to verify integrity

## Integration Notes
To use the audit service in other parts of the codebase:

```rust
use ramp_core::service::ComplianceAuditService;
use ramp_core::repository::ComplianceEventType;

// Log a KYC tier change
audit_service.log_kyc_tier_change(
    &tenant_id,
    "user123",
    1, // old tier
    2, // new tier
    "KYC documents verified",
    AuditContext::admin("admin@example.com"),
).await?;
```
