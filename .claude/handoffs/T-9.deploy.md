## Deliverables

### 1. Deployment Scripts (`contracts/script/`)
- `DeployLayers.s.sol`: Generic deployment script for any supported EVM chain.
- `DeployBase.s.sol`: Base-specific deployment (Mainnet & Sepolia).
- `DeployArbitrum.s.sol`: Arbitrum-specific deployment (One & Sepolia).
- `DeployOptimism.s.sol`: Optimism-specific deployment (Mainnet & Sepolia).
- `DeployPolygonZkEVM.s.sol`: Polygon zkEVM-specific deployment (Mainnet & Cardona) with custom gas estimation.

### 2. Configuration (`contracts/config/`)
JSON configuration files created for all networks:
- **Base**: `base-mainnet.json`, `base-sepolia.json`
- **Arbitrum**: `arbitrum-one.json`, `arbitrum-sepolia.json`
- **Optimism**: `optimism-mainnet.json`, `optimism-sepolia.json`
- **Polygon zkEVM**: `polygon-zkevm-mainnet.json`, `polygon-zkevm-cardona.json`

Each config includes:
- Chain IDs & RPC URLs
- Explorer APIs & Verification keys
- ERC-4337 EntryPoint & Bundler endpoints
- Token addresses (USDC, WETH, etc.)

### 3. Orchestration
- `contracts/deploy-all.sh`: Master shell script to deploy to all layers in sequence.
  - Usage: `./contracts/deploy-all.sh`
  - Requires `.env` with API keys (`DEPLOYER_PRIVATE_KEY`, `BASESCAN_API_KEY`, etc.)

## Usage Guide

### Single Chain Deployment
```bash
# Example: Base Sepolia
forge script script/DeployBase.s.sol:DeployBase --rpc-url base_sepolia --broadcast --verify

# Example: Polygon zkEVM Cardona
forge script script/DeployPolygonZkEVM.s.sol:DeployPolygonZkEVMTestnet --rpc-url polygon_zkevm_cardona --broadcast --verify
```

### Multi-Chain Deployment
```bash
# Deploy to all testnets defined in the script
./contracts/deploy-all.sh
```

## Notes for Next Phase
- Ensure `.env` is populated with all necessary API keys before running `deploy-all.sh`.
- The `DeployPolygonZkEVM` script applies a 1.2x gas multiplier for zk-proof overhead.
- All scripts support both EntryPoint v0.6 and v0.7 (configurable via `USE_ENTRYPOINT_V7` env var).
