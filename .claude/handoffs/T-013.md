# Task Handoff: Tenant Onboarding Flow (T-013)

## Summary
Implemented the tenant onboarding flow including the `OnboardingService` and related API endpoints. This allows administrators to create new tenants, manage their configuration, and handle their lifecycle (activation, suspension).

## Changes
- Created `crates/ramp-core/src/service/onboarding.rs`: Core logic for tenant creation, API key generation, and lifecycle management.
- Created `crates/ramp-api/src/handlers/admin/onboarding.rs`: API handlers for the onboarding endpoints.
- Updated `crates/ramp-core/src/service/mod.rs`: Exported `OnboardingService`.
- Updated `crates/ramp-api/src/handlers/admin/mod.rs`: Exported onboarding handlers.
- Updated `crates/ramp-api/src/router.rs`: Registered the new admin routes.
- Updated `crates/ramp-api/src/main.rs`: Instantiated and wired `OnboardingService`.

## API Endpoints
- `POST /v1/admin/tenants`: Create a new tenant
- `PATCH /v1/admin/tenants/:id`: Update tenant config (webhooks, limits)
- `POST /v1/admin/tenants/:id/api-keys`: Regenerate API keys
- `POST /v1/admin/tenants/:id/activate`: Activate a tenant
- `POST /v1/admin/tenants/:id/suspend`: Suspend a tenant
