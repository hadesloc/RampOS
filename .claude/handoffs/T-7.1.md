# T-7.1: Licensing Requirements API - Handoff

## Summary
Implemented Vietnam licensing requirements API endpoints for tracking regulatory compliance and deadlines per exchange tenant.

## Files Created

### Types (ramp-common)
- `crates/ramp-common/src/licensing.rs` - Domain types:
  - `LicenseRequirementId`, `LicenseSubmissionId` - ID types
  - `LicenseStatus` - Pending, Submitted, Approved, Expired, Rejected, UnderReview
  - `LicenseType` - SBV_PAYMENT_LICENSE, AML_REGISTRATION, DATA_PROTECTION, BUSINESS_REGISTRATION, CRYPTO_LICENSE, FOREX_LICENSE
  - `LicenseRequirement`, `TenantLicenseStatus`, `LicenseSubmission`, `SubmittedDocument`, `LicenseDeadline`

### Repository (ramp-core)
- `crates/ramp-core/src/repository/licensing.rs` - Database layer:
  - `LicenseRequirementRow`, `TenantLicenseStatusRow`, `LicenseSubmissionRow`
  - `LicensingRepository` trait with methods:
    - `list_requirements()`, `get_requirement()`
    - `get_tenant_license_statuses()`, `upsert_tenant_license_status()`
    - `create_submission()`, `list_submissions()`, `update_submission_status()`
    - `get_upcoming_deadlines()`, `count_requirements()`
  - `PgLicensingRepository` PostgreSQL implementation

### API Handlers (ramp-api)
- `crates/ramp-api/src/handlers/admin/licensing.rs` - Endpoints:
  - `GET /v1/admin/licensing/requirements` - List all licensing requirements
  - `GET /v1/admin/licensing/status/:tenant_id` - Get tenant license status overview
  - `POST /v1/admin/licensing/submit` - Submit license documents
  - `GET /v1/admin/licensing/deadlines` - Get upcoming/overdue deadlines

### Database Migration
- `migrations/015_licensing_requirements.sql`:
  - `license_requirements` table with Vietnam-specific seed data
  - `tenant_license_status` table (tenant isolation via RLS)
  - `license_submissions` table (document tracking)
  - Proper indexes and RLS policies

## API Endpoints

### GET /v1/admin/licensing/requirements
Lists all Vietnam licensing requirements.

**Query params:**
- `limit` (default: 20)
- `offset` (default: 0)

**Response:**
```json
{
  "data": [{
    "id": "lic_sbv_payment",
    "name": "SBV Payment Service License",
    "licenseType": "SBV_PAYMENT_LICENSE",
    "regulatoryBody": "State Bank of Vietnam",
    "deadline": "2026-06-01T00:00:00Z",
    "requiredDocuments": ["Business registration certificate", ...],
    "isMandatory": true
  }],
  "total": 5,
  "limit": 20,
  "offset": 0
}
```

### GET /v1/admin/licensing/status/:tenant_id
Gets tenant's compliance overview across all requirements.

**Response:**
```json
{
  "tenantId": "tenant_123",
  "totalRequirements": 5,
  "approvedCount": 2,
  "pendingCount": 2,
  "expiredCount": 1,
  "licenses": [{
    "requirementId": "lic_sbv_payment",
    "requirementName": "SBV Payment Service License",
    "status": "APPROVED",
    "licenseNumber": "SBV-2026-001234",
    "expiryDate": "2027-01-15T00:00:00Z"
  }]
}
```

### POST /v1/admin/licensing/submit
Submits documents for a licensing requirement.

**Request:**
```json
{
  "requirementId": "lic_sbv_payment",
  "documents": [{
    "name": "Business Registration Certificate",
    "fileUrl": "https://storage.example.com/docs/brc.pdf",
    "fileHash": "sha256:abc123...",
    "fileSizeBytes": 1024000
  }],
  "submittedBy": "admin@example.com"
}
```

### GET /v1/admin/licensing/deadlines
Gets upcoming and overdue licensing deadlines.

**Query params:**
- `daysAhead` (default: 30)

**Response:**
```json
{
  "upcoming": [{
    "requirementId": "lic_aml_reg",
    "requirementName": "AML Registration",
    "deadline": "2026-03-15T00:00:00Z",
    "daysRemaining": 37,
    "status": "PENDING",
    "isOverdue": false
  }],
  "overdue": [{
    "requirementId": "lic_business_reg",
    "daysRemaining": -5,
    "isOverdue": true
  }]
}
```

## Seeded Requirements
The migration seeds 5 Vietnam-specific licensing requirements:
1. SBV Payment Service License (mandatory)
2. AML Registration (mandatory)
3. Data Protection Registration (mandatory)
4. Business Registration Certificate (mandatory)
5. Foreign Exchange License (optional)

## Integration Notes
- `AppState.licensing_repo` is `Option<Arc<dyn LicensingRepository>>` - initialize with `PgLicensingRepository::new(pool.clone())`
- All endpoints require admin API key authentication
- RLS ensures tenant data isolation
- Document URLs should be pre-signed storage URLs
