### Notes
- Background workers using the `rampos` user MUST set `app.current_tenant` before querying tenant-specific data, OR use the `rampos_system` role (which has `BYPASSRLS`).
- The `rampos_system` role was created in migration `008_add_missing_rls.sql` and granted to `rampos`. However, the application must explicitly `SET ROLE rampos_system` or use a separate connection pool with that user to bypass RLS.
- Current codebase inspection shows no explicit `SET ROLE` calls, suggesting the app relies on setting `app.current_tenant` for all operations.
