# Handoff Report: Device/IP Anomaly & Load Testing (T-6.2.4 + T-4.3.3)

## 1. Device/IP Anomaly Detection (T-6.2.4)
Implemented `DeviceAnomalyRule` in `crates/ramp-compliance/src/aml/device_anomaly.rs`.

### Features
- **Suspicious Country Check**: Flags transactions from high-risk countries (NK, IR, SY, CU).
- **Max Devices Check**: Limits unique devices per user (default 5).
- **Max IPs Check**: Limits unique IPs per day (default 5).
- **Impossible Travel**: Detects physically impossible travel speed (>1000km/h).
- **VPN/Proxy Detection**: Flags known VPN/Proxy usage.
- **Device Sharing**: Detects if a device is used by multiple users.

### Usage
```rust
let rule = DeviceAnomalyRule::new(store);
// or with custom config
let rule = DeviceAnomalyRule::with_config(store, 3, 5, vec!["US".into()]);
```

## 2. Load Testing Suite (T-4.3.3)
Enhanced k6 load testing suite in `tests/load/`.

### Components
- `config.js`: Centralized config, thresholds, and scenarios.
- `scenarios.js`: Reusable load profiles (smoke, load, stress, spike).
- `trade.js`: New trade endpoint test.
- `mixed.js`: Combined traffic simulation (40% payin, 30% payout, 30% trade).
- `run-load-tests.sh`: Automation script.

### How to Run
```bash
# Run all tests
./tests/load/run-load-tests.sh

# Run specific scenario
SCENARIO=stress ./tests/load/run-load-tests.sh
```
