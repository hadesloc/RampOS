# T-022: Add OpenTelemetry Tracing

## Status
Completed

## Changes
1.  **Rust Integration**:
    - Updated `crates/ramp-common/Cargo.toml` to include `grpc-tonic` feature for `opentelemetry-otlp`.
    - Modified `crates/ramp-common/src/telemetry.rs` to use `tonic()` (gRPC) exporter on port 4317.
    - Verified `telemetry.rs` initializes `tracing` with both JSON logging and OpenTelemetry layers.

2.  **Instrumentation**:
    - Added `#[tracing::instrument]` to `ramp-api/src/handlers/intent.rs` (Intent handlers).
    - Added `#[tracing::instrument]` to `ramp-core/src/repository/intent.rs` (Intent repository methods).
    - This covers the critical path for Intent processing (API -> DB).

3.  **Infrastructure (K8s)**:
    - Created `k8s/base/jaeger.yaml`: Deploys Jaeger All-in-One with OTLP enabled.
    - Created `k8s/base/otel-collector.yaml`: Configures OTEL Collector to receive OTLP (gRPC/HTTP) and export to Jaeger.
    - Updated `k8s/base/kustomization.yaml` to include new resources.
    - Updated `k8s/base/configmap.yaml` to set `OTEL_EXPORTER_OTLP_ENDPOINT` to `http://otel-collector:4317` and `OTEL_SERVICE_NAME` to `rampos-api`.

## Verification
- `cargo check` passes for all crates.
- `otel-collector` is configured to listen on 0.0.0.0:4317 (gRPC) and 0.0.0.0:4318 (HTTP).
- `ramp-common` points to `http://otel-collector:4317` (via env var default or k8s service). Note: The default in code is `None` for endpoint, so `OTEL_EXPORTER_OTLP_ENDPOINT` must be set in deployment (usually `http://otel-collector:4317`).

## Testing Instructions
1.  **Deploy**:
    ```bash
    kubectl apply -k k8s/overlays/dev
    ```
2.  **Run Traffic**:
    - Create an intent via API.
    - List intents.
3.  **Verify Traces**:
    - Port-forward Jaeger: `kubectl port-forward svc/jaeger 16686:16686`
    - Open `http://localhost:16686`
    - Select `rampos-api` (or configured service name) and find traces.
    - You should see spans for `get_intent`, `list_intents`, and repository calls like `get_by_id`.

## Configuration
- Ensure `OTEL_EXPORTER_OTLP_ENDPOINT` is set to `http://otel-collector:4317` in the application deployment (already standard for OTLP gRPC).
