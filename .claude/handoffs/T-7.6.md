# T-7.6: Regulatory Webhook Notifications - Handoff

## Status: COMPLETED

## Summary
Implemented regulatory webhook notifications for license deadlines and compliance alerts.

## Changes Made

### 1. New Webhook Events (`crates/ramp-core/src/event.rs`)
Added `RegulatoryEvent` enum with 4 event types:
- `license_expiring` - License expiration notifications (7/30 days)
- `document_required` - New document requirement notifications
- `compliance_alert` - Compliance threshold breach alerts
- `sbv_submission_due` - SBV report deadline notifications

Added `AlertSeverity` enum: Low, Medium, High, Critical

Extended `EventPublisher` trait with `publish_regulatory_event()` method.
Implemented for both `NatsEventPublisher` and `InMemoryEventPublisher`.

### 2. License Deadline Checker (`crates/ramp-core/src/jobs/license_deadline_checker.rs`)
- Daily scheduled job to check for expiring licenses
- Sends notifications at 7-day and 30-day marks
- Uses `LicenseRepository::find_expiring_before()` for queries
- Includes unit tests with mock repository

### 3. Compliance Alert Scheduler (`crates/ramp-core/src/jobs/compliance_alert_scheduler.rs`)
- Hourly scheduled job for compliance monitoring
- `check_compliance_thresholds()` - Monitors limit breaches with severity levels
- `check_sbv_deadlines()` - Sends notifications at 14, 7, 3, 1 day marks
- Includes unit tests with mock repository

### 4. Repository Updates

#### `crates/ramp-core/src/repository/compliance.rs` (NEW)
- `ComplianceBreach` struct for threshold alerts
- `SbvReportSchedule` struct for SBV deadlines
- `ComplianceRepository` trait

#### `crates/ramp-core/src/repository/license.rs`
Added `LicenseRow` struct and methods:
- `find_expiring_before()` - Query licenses by expiry date
- `get_by_id()`, `list_by_tenant()`, `create()`, `update()`

#### `crates/ramp-core/src/jobs/mod.rs`
Exported new job modules and structs.

## Webhook Payload Format
```json
{
  "tenant_id": "tenant_123",
  "event": {
    "event": "license_expiring",
    "data": {
      "license_id": "lic_1",
      "license_type": "exchange",
      "expires_at": "2026-03-15T00:00:00Z",
      "days_remaining": 30
    }
  },
  "timestamp": "2026-02-06T20:30:00Z"
}
```

## Integration Notes
1. Both jobs are designed to run as background tasks using tokio intervals
2. License checker runs daily (86400s), compliance checker runs hourly (3600s)
3. Jobs integrate with existing webhook delivery system via EventPublisher
4. All events are tenant-scoped for multi-tenant support
