## Overview

This task implements complete API integration for the RampOS frontend portal, connecting all user-facing pages with backend services for authentication, KYC, wallet management, and transactions.

## Files Created

### 1. Portal API Client
**File**: `frontend/src/lib/portal-api.ts`

A comprehensive API client for all portal endpoints:
- Authentication APIs (passkey, magic link)
- User management APIs
- KYC submission and status APIs
- Wallet management APIs (balances, deposit info, create wallet)
- Transaction APIs (list, create deposit/withdraw, confirm)

Key features:
- Token-based authentication with localStorage persistence
- Custom error handling with PortalApiError class
- Type-safe interfaces for all API responses
- Pagination support for transaction listing

### 2. Auth Context Provider
**File**: `frontend/src/contexts/auth-context.tsx`

React Context for managing authentication state across the portal:
- Provides: user, wallet, isLoading, isAuthenticated, error
- Methods: loginWithPasskey, registerWithPasskey, loginWithMagicLink, verifyMagicLink, logout, refreshWallet, createWallet
- Persists auth tokens in localStorage
- Auto-fetches user session on mount
- WebAuthn credential serialization helpers (base64UrlEncode/base64UrlDecode)

## Architecture Decisions

1. **Separate Portal API Client**: Created `portal-api.ts` separate from admin `api.ts` to maintain clean separation between user-facing and admin endpoints.

2. **Auth Context Pattern**: Used React Context for auth state to avoid prop drilling and enable easy access to auth state across all portal pages.

3. **Protected Route Pattern**: Each page checks authentication and redirects to login if not authenticated.

4. **Wallet-First Design**: Pages that require wallet (deposit, withdraw, assets) show wallet creation prompt if wallet doesn't exist.

5. **WebAuthn with Fallback**: Primary authentication is passkey/WebAuthn with magic link as fallback for unsupported browsers.
