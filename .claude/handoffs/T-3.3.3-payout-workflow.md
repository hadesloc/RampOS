# Task T-3.3.3: Payout Workflow Implementation

## Summary
Implemented the full Payout Workflow using a dedicated `PayoutWorkflow` struct and Temporal worker integration.

## Changes
1.  **New File**: `crates/ramp-core/src/workflows/payout.rs`
    -   `PayoutWorkflow` struct orchestrating the payout process.
    -   Activities: `validate_balance`, `run_aml_check`, `hold_funds`, `submit_to_bank`, `wait_for_settlement` (callback), `finalize_settlement`, `reverse_funds`.
    -   Structs: `PayoutWorkflowInput`, `PayoutWorkflowResult`, `BankAccountInfo`, `SettlementResult`.

2.  **Refactoring**: `crates/ramp-core/src/workflows/mod.rs`
    -   Exported `payout` module.
    -   Removed duplicate struct definitions (re-exported from `payout`).
    -   Removed legacy `payout_activities` module.

3.  **Update**: `crates/ramp-core/src/temporal_worker.rs` and `crates/ramp-core/src/workflows/worker.rs`
    -   Updated `TemporalWorker` to include `AmlEngine` dependency.
    -   Updated `execute_payout_workflow` to instantiate and run `PayoutWorkflow`.
    -   Updated tests to provide `AmlEngine` and `CaseManager`.

## Features
-   **Balance Validation**: Checks user balance before processing.
-   **AML Integration**: Runs transactions through `AmlEngine` before approval.
-   **Ledger Integration**: Holds funds (Debit User) upon approval, releases or reverses based on settlement outcome.
-   **State Management**: Updates Intent state at each step (POLICY_APPROVED, FUNDS_HELD, SUBMITTED_TO_BANK, COMPLETED, REJECTED_*).
-   **Settlement Handling**: Waits for bank settlement signal with timeout handling.
