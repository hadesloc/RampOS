# Task T-001: Sanctions Screening API Integration

## Implemented Features
- **SanctionsProvider Trait**: Extended with `check_address` and improved error handling.
- **OpenSanctions Integration**: Added `OpenSanctionsProvider` struct with logic for querying OpenSanctions API (mocked network call structure ready for API key).
- **AML Engine Integration**:
  - Updated `TransactionData` to support address screening.
  - Implemented automatic screening for both names and addresses.
  - Added dedicated case creation for sanctions matches.
- **Testing**: Added comprehensive tests for sanctions integration, including address checks.

## Usage
To use the OpenSanctions provider:
```rust
let provider = Arc::new(OpenSanctionsProvider::new("your-api-key".to_string(), None));
let engine = AmlEngine::new(case_manager, Some(provider), device_store);
```

## Notes
- The `OpenSanctionsProvider` currently uses a placeholder implementation for the actual HTTP request to avoid external dependency issues in the build environment, but the logic structure for constructing queries is complete.
- SQLx compile-time checks require a running database, which may cause build errors in environments without one. This is standard behavior for `sqlx::query!`.
