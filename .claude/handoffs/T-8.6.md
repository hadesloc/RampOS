## Deliverables

### 1. VNST Protocol Service (`crates/ramp-core/src/stablecoin/vnst_protocol.rs`)

Core protocol implementation with:
- **VnstProtocolConfig**: Configurable mint/burn limits, fees, peg thresholds
- **VnstMintRequest/Response**: Mint VNST with VND deposit
- **VnstBurnRequest/Response**: Burn VNST for VND withdrawal
- **VnstReserveInfo**: Reserve breakdown with collateralization ratio
- **VnstPegStatus**: Real-time peg health monitoring
- **VnstProtocolDataProvider trait**: Abstraction for data access
- **MockVnstProtocolDataProvider**: Testing implementation

Key features:
- VND/VNST 1:1 peg with 18 decimals
- Configurable mint/burn fees (default 0.1%)
- Peg deviation monitoring with warning (0.5%) and critical (2%) thresholds
- Operations suspended when peg deviation is critical
- Reserve proof attestation support
- Collateralization ratio tracking

### 2. Module Export (`crates/ramp-core/src/stablecoin/mod.rs`)

### 3. API Handlers (`crates/ramp-api/src/handlers/stablecoin.rs`)

New API endpoints:
- `POST /v1/stablecoin/vnst/mint` - Mint VNST with VND
- `POST /v1/stablecoin/vnst/burn` - Burn VNST for VND
- `GET /v1/stablecoin/vnst/reserves` - Get reserve information
- `GET /v1/stablecoin/vnst/peg` - Get peg status
- `GET /v1/stablecoin/vnst/config` - Get protocol configuration

## API Reference

### Mint VNST
```http
POST /v1/stablecoin/vnst/mint
Content-Type: application/json
X-API-Key: <api_key>

{
  "vndAmount": "1000000",

  "recipientAddress": "0x...",
  "idempotencyKey": "optional-key"
}
```

### Burn VNST
```http
POST /v1/stablecoin/vnst/burn
Content-Type: application/json
```

