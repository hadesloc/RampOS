# Handoff: Payin Workflow Implementation

## Task Overview
Implemented `PayinWorkflow` for handling VND pay-in transactions using Temporal workflow patterns.

## Deliverables
- `crates/ramp-core/src/workflows/payin.rs`: Workflow implementation including activities and signal handling.
- `crates/ramp-core/src/workflows/mod.rs`: Exposed `PayinWorkflow` module.
- `crates/ramp-core/src/workflows/worker.rs`: Updated worker to use the new `PayinWorkflow` struct instead of inline logic.

## Technical Details
- **Workflow Steps:**
    1. **Issue Instruction:** Calls activity to generate reference code.
    2. **Wait for Signal:** Waits for `BankConfirmation` signal (with 24h timeout).
    3. **Credit Balance:** Calls activity to credit user ledger.
    4. **Webhook:** Calls activity to notify external system.
- **State Management:**
    - Updates intent state to `INSTRUCTION_ISSUED` after creation.
    - Updates to `COMPLETED` on success.
    - Updates to `EXPIRED` on timeout.
- **Signal Handling:**
    - Uses a closure-based signal provider to allow testing without actual Temporal server.
    - Integrated with `TemporalWorker`'s signal buffer.

## Testing
- Unit tests added in `crates/ramp-core/src/workflows/payin.rs` covering:
    - Happy path (successful confirmation).
    - Timeout path (expiration).
- *Note:* Tests skipped in CI environment due to missing `dlltool` (Windows/Rust environment issue), but code is standard Rust/Tokio async logic.
