## Integration Notes

### AAServiceState
The `AAServiceState` contains:
- `SmartAccountService` - Creates and manages smart accounts
- `BundlerClient` - Communicates with ERC-4337 bundler
- `PaymasterService` - Handles gas sponsorship
- `ChainConfig` - Chain-specific configuration

### Enabling AA Routes
To enable AA routes, set `aa_service` in `AppState`:
```rust
let aa_state = AAServiceState::new(ChainConfig::ethereum_mainnet());
let app_state = AppState {
    // ... other services
    aa_service: Some(aa_state),
};
```

### Authentication
All AA endpoints require authentication via the existing `auth_middleware`. They use the `TenantContext` extension to verify tenant ownership.

### Sponsorship
The `send_user_operation` endpoint supports optional sponsorship via the `sponsor: bool` field. When true:
1. Checks sponsorship policy via `PaymasterService::can_sponsor`
2. Generates paymaster data via `PaymasterService::sponsor`
3. Attaches paymaster data to the UserOperation
