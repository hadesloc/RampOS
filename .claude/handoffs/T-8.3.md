Integrate Stargate/Across for cross-chain stablecoin transfers.

## Status: COMPLETED

## Implementation Details

### 1. Bridge Module (`crates/ramp-core/src/bridge/`)

Created new module with three files:

#### `mod.rs` - Core Types and Registry
- `CrossChainBridge` trait - Unified interface for all bridge providers
- `BridgeRegistry` - Manages multiple bridge providers
- `BridgeConfig` - Configuration for routers and spoke pools
- Types:
  - `BridgeQuote` - Quote for cross-chain transfer
  - `BridgeStatus` - Transfer status tracking
  - `BridgeTransfer` - Transfer record
  - `BridgeToken` - USDT, USDC enum
  - `SupportedChain` - Ethereum, Arbitrum, Base, Optimism, Polygon

#### `stargate.rs` - Stargate V2 Integration
- `StargateBridge` implementation
- LayerZero Endpoint IDs mapping
- Stargate Pool IDs for tokens
- Fee calculation (0.06% bridge fee)
- Gas estimation per chain

#### `across.rs` - Across Protocol Integration
- `AcrossBridge` implementation
- SpokePool integration
- Relayer fee calculation
- LP fee calculation
- Faster finality (30s between L2s)

### 4. Bridge Trait Interface

```rust
#[async_trait]
pub trait CrossChainBridge: Send + Sync {
    fn name(&self) -> &str;
    fn supported_chains(&self) -> Vec<ChainId>;
    fn supports_route(&self, from_chain: ChainId, to_chain: ChainId, token: BridgeToken) -> bool;
    async fn quote(
        &self,
        from_chain: ChainId,
        to_chain: ChainId,
        token: Address,
        amount: U256,
        recipient: Address,
    ) -> Result<BridgeQuote>;
    async fn bridge(&self, quote: BridgeQuote) -> Result<TxHash>;
    async fn status(&self, tx_hash: TxHash) -> Result<BridgeStatus>;
    fn estimated_time(&self, from_chain: ChainId, to_chain: ChainId) -> u64;
}
```

### 6. Key Features

1. **Multi-bridge support** - Registry manages multiple bridges, can get best quote
2. **Quote comparison** - Get quotes from all bridges, sorted by output amount
3. **Route validation** - Checks if route is supported before quoting
4. **Fee breakdown** - Separates bridge fee and gas fee
5. **Time estimation** - Different estimates per route (L2-L2 faster than L1)
6. **Status tracking** - Track transfer through Pending -> InProgress -> Completed
