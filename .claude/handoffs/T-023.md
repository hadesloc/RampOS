# T-023: Add Log Aggregation Handoff

## Deliverables
1. **Loki Deployment**: `k8s/base/loki.yaml`
   - Configured with filesystem storage
   - Retention set to 24h
   - No authentication (internal use)

2. **Promtail DaemonSet**: `k8s/base/promtail.yaml`
   - Scrapes all pod logs
   - Enriches with Kubernetes metadata (namespace, pod, container)
   - Forwards to Loki

3. **Grafana Dashboards**: `k8s/monitoring/grafana-dashboards/logs.json`
   - Log volume overview
   - Searchable log stream
   - Error rate tracking

4. **Application Configuration**:
   - `k8s/base/configmap.yaml` updated to enable JSON logs
   - `crates/ramp-common/src/telemetry.rs` verified (already supports JSON logging)

## Usage
1. Apply changes:
   ```bash
   kubectl apply -k k8s/monitoring
   kubectl apply -k k8s/base
   ```

2. View logs:
   - Access Grafana
   - Navigate to "Application Logs" dashboard
   - Filter by application label (e.g., `rampos-api`)

## Verification
- Check Loki pod status: `kubectl get pods -l app=loki`
- Check Promtail pods: `kubectl get pods -l app=promtail`
- Verify logs in Grafana Explore view (DataSource: Loki)
