# T-7.2: Compliance Document Generator - Handoff

## Task Summary
Auto-generate required compliance documents for State Bank of Vietnam (SBV) submission.

## Status: COMPLETED

## Implementation Details

### 1. Document Generator Module (`crates/ramp-compliance/src/documents/`)

Created new module with two files:

#### `mod.rs` - Main Generator Implementation
- `ComplianceDocumentGenerator` struct with methods:
  - `generate_compliance_report()` - Full SBV compliance report
  - `generate_transaction_summary()` - Transaction volumes by type (payin, payout, trade)
  - `generate_kyc_statistics()` - KYC tier completion stats
  - `generate_aml_metrics()` - AML compliance metrics and alert rates
  - `export_to_html()` - Vietnamese HTML export for SBV submission
  - `export_to_json()` - JSON export
  - `export_to_csv()` - CSV export
  - `save_document()` - Save to DocumentStorage

#### `types.rs` - Type Definitions
- `ComplianceReportType` - Report type enum (FullCompliance, TransactionSummary, etc.)
- `DocumentFormat` - Output format enum (Html, Pdf, Json, Csv)
- `DocumentStatus` - Generation status
- `CompanyInfo` - Tenant registration info
- `TransactionSummary` - Transaction metrics by type
- `KYCStatistics` - KYC tier distribution and completion rates
- `AMLMetrics` - AML alert counts and resolution rates
- `RegulatorySubmissions` - SAR/CTR submission history
- `ComplianceReport` - Full report combining all sections
- `GeneratedDocument` - Document metadata after saving

### 2. API Handlers (`crates/ramp-api/src/handlers/admin/documents.rs`)

New admin endpoints:

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/admin/documents/generate` | POST | Generate and save compliance document |
| `/v1/admin/documents/:id` | GET | Download document by ID |
| `/v1/admin/documents` | GET | List generated documents |
| `/v1/admin/documents/compliance-report` | POST | Generate and download report directly |
| `/v1/admin/documents/transaction-summary` | GET | Get transaction summary JSON |
| `/v1/admin/documents/kyc-statistics` | GET | Get KYC statistics JSON |
| `/v1/admin/documents/aml-metrics` | GET | Get AML metrics JSON |

### 3. Report Content

The compliance report includes:

1. **Company Registration Info** (from tenant config)
   - Company name, Tax ID, Business registration number
   - Address, License number

2. **Transaction Volumes by Type**
   - Payin count and volume (VND)
   - Payout count and volume (VND)
   - Trade count and volume (VND)
   - Daily breakdown

3. **KYC Completion Rates by Tier**
   - Total users by tier (tier_0 through tier_3)
   - Approved/Rejected/Pending/Expired counts
   - Completion rate percentage
   - Average verification time

4. **AML Compliance Metrics**
   - Alert counts by severity (Critical, High, Medium, Low)
   - Resolution rate percentage
   - Average resolution time
   - Alerts by rule

5. **SAR/CTR Submission History**
   - SAR filed count (REPORTED cases)
   - CTR filed count (transactions >= 200M VND)
