# Task T-8.1: Multi-Stablecoin Support - Handoff

## Status: COMPLETED

## Summary
Implemented multi-stablecoin support with unified interface for USDT, USDC, DAI, and VNST.

## Files Created

### Stablecoin Module (`crates/ramp-core/src/stablecoin/`)
1. **`mod.rs`** - Core module with:
   - `Stablecoin` trait - unified interface for all stablecoins
   - `StablecoinRegistry` - manages all supported tokens
   - `TenantTokenConfig` - per-tenant configuration
   - `StablecoinMetadata` - token metadata (name, decimals, logo)
   - `TokenDeployment` - chain-specific contract addresses

2. **`usdt.rs`** - USDT (Tether) implementation
   - 6 decimals
   - Contract addresses for Ethereum, Polygon, BSC, Arbitrum, Optimism, Base

3. **`usdc.rs`** - USDC (Circle) implementation
   - 6 decimals
   - Native USDC contract addresses for major chains

4. **`dai.rs`** - DAI (MakerDAO) implementation
   - 18 decimals
   - Multi-chain support

5. **`vnst.rs`** - VNST (Vietnam Stablecoin) implementation
   - 18 decimals
   - VND conversion utilities
   - Primarily BSC-focused

### Database Migration
- **`migrations/016_multi_stablecoin.sql`**
  - `supported_tokens` table - token configurations per tenant
  - `token_chain_deployments` table - contract addresses per chain
  - `token_balances` table - user token balances
  - `token_transactions` table - on-chain transaction history
  - `user_token_balances` view - aggregated balances
  - RLS policies for tenant isolation
  - Performance indexes

## Key Features

### Stablecoin Trait
```rust
#[async_trait]
pub trait Stablecoin: Send + Sync {
    fn symbol(&self) -> &str;
    fn decimals(&self) -> u8;
    fn contract_address(&self, chain_id: u64) -> Option<Address>;
    async fn balance_of(&self, chain_id: u64, address: Address) -> Result<U256>;
    async fn transfer(&self, chain_id: u64, from: Address, to: Address, amount: U256) -> Result<TxHash>;
}
```

### Token Registry
```rust
let registry = StablecoinRegistry::new();

// Get token by symbol
let usdt = registry.get_token("USDT").unwrap();

// Get tokens supported on chain
let eth_tokens = registry.tokens_on_chain(1);

// Configure per-tenant
registry.configure_token_for_tenant("USDT", tenant_id, config)?;

// Validate deposit/withdraw
registry.validate_deposit("USDT", &tenant_id, 1, amount)?;
```

### Per-Tenant Configuration
```rust
TenantTokenConfig {
    symbol: "USDT".to_string(),
    enabled: true,
    min_deposit: U256::from(1_000_000u64), // 1 USDT
    max_deposit: Some(U256::from(1_000_000_000_000u64)),
    deposit_fee_bps: 0,
    withdraw_fee_bps: 10,
    allowed_chains: vec![1, 137, 56],
}
```

## Integration Points

### With Smart Account Service (ramp-aa)
- Use `SmartAccountService::build_transfer_op()` for token transfers
- Token operations go through ERC-4337 UserOperation

### With Deposit Service (ramp-core)
- `CreateDepositRequest.symbol` now supports all stablecoins
- Token validation via `StablecoinRegistry::validate_deposit()`

### With Ledger (ramp-ledger)
- `LedgerCurrency::from_symbol()` for multi-token ledger entries
