# Task 6.2.4: Device/IP Anomaly Detection Handoff

## Implemented Features
- **DeviceAnomalyRule**: New AML rule that checks for:
  - New device usage
  - Known VPN/Proxy usage
  - Country changes
  - Impossible travel (speed > 1000km/h)
  - Multiple users sharing same device
- **DeviceHistoryStore**: Trait for storing user device history (devices, IPs, locations).
- **MockDeviceHistoryStore**: In-memory implementation for testing.
- **DeviceContext**: Data structure for passing device metadata to rules.
- **RuleContext Update**: Added `metadata` field to support device context.

## Files Created/Modified
- `crates/ramp-compliance/src/aml/device_anomaly.rs` (New)
- `crates/ramp-compliance/src/aml.rs` (Modified)
- `crates/ramp-compliance/src/rules.rs` (Modified)

## Usage
To use the new rule, instantiate `DeviceAnomalyRule` with a `DeviceHistoryStore` implementation and add it to `AmlEngine`.

```rust
let store = Arc::new(MockDeviceHistoryStore::new()); // Or real impl
let rule = Box::new(DeviceAnomalyRule::new(store));
// Add to engine...
```

The `AmlEngine::check_transaction` now accepts `metadata` in `TransactionData`. The metadata should include a "device" field with `DeviceContext` structure.

```json
{
  "device": {
    "device_id": "...",
    "device_fingerprint": "...",
    "ip_address": "...",
    "country": "VN",
    ...
  }
}
```
