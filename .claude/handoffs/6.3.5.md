# Task 6.3.5: Add score history tracking - Handoff

## Implementation Details
- **Database Schema**:
  - `user_id` and `intent_id` are stored as `VARCHAR` to support the string-based IDs used in the application (`UserId` and `IntentId`).
  - `triggered_rules` is stored as `JSONB`.
  - Indexes created for efficient querying by user and time.

- **Trend Detection**:
  - Implemented `detect_trend` which fetches recent history and calculates trend based on score movement.
  - Logic is unit tested via `calculate_trend_internal`.
