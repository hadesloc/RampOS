# Handoff: T-2.2.2 Timestamp Validation

## Summary
Implemented validation for the `X-Timestamp` header in the authentication middleware to prevent replay attacks and ensure request freshness.

## Changes
- Modified `crates/ramp-api/src/middleware/auth.rs`
- Added `validate_timestamp` function to check for drift against server time
- Configured 5-minute (300 seconds) drift tolerance
- Added unit tests for ISO8601 and Unix timestamp formats

## Verification
- Unit tests added in `middleware/auth.rs` covering:
  - Valid ISO8601 timestamp
  - Future ISO8601 timestamp (drift > 5m)
  - Valid Unix timestamp (seconds)
  - Valid Unix timestamp (milliseconds)
  - Missing header

## Notes
- Validation happens *before* API key lookup to fail fast
- Supports both ISO8601 and Unix timestamps (auto-detected)
- Returns HTTP 401 if timestamp is missing or expired
