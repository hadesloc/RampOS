# T-7.3: SBV Reporting Format - Handoff

## Status: COMPLETED

## Summary
Implemented State Bank of Vietnam (SBV) reporting format for CTR (Currency Transaction Reports) and SAR (Suspicious Activity Reports) compliant with Circular 09/2023/TT-NHNN.

## Files Created

### 1. `crates/ramp-compliance/src/reports/sbv_ctr.rs`
SBV-compliant Currency Transaction Report implementation:
- `SbvCtrReport` - Main CTR report structure
- `SbvCtrTransaction` - Individual transaction data
- `SbvCustomerInfo` - Customer identification details (CCCD, CMND, Passport)
- `SbvFilingInstitution` - Reporting institution data
- `SbvTransactionType` - Transaction classifications with SBV codes
- `SbvIdType` - ID document types with SBV-standard codes
- `to_sbv_xml()` - XML export in SBV format
- `SBV_CTR_THRESHOLD_VND` constant (300M VND)

### 2. `crates/ramp-compliance/src/reports/sbv_sar.rs`
SBV-compliant Suspicious Activity Report implementation:
- `SbvSarReport` - Main SAR report structure
- `SbvSarSubject` - Subject under investigation
- `RiskIndicator` - Detected risk indicators
- `TransactionPattern` - Pattern analysis data
- `SuspiciousActivityType` - Activity classifications (Structuring, AML, etc.)
- `SbvRiskLevel` - Risk levels (Critical, High, Medium, Low)
- `RecommendedAction` - Actions (Freeze, Restrict, Report to Law Enforcement)
- `SbvSarWeeklySummary` - Weekly aggregated summary
- `to_sbv_xml()` - XML export in SBV format

### 3. `crates/ramp-compliance/src/reports/sbv_scheduler.rs`
Automatic report scheduling:
- `SbvReportScheduler` - Main scheduler with configurable timing
- `SbvScheduleConfig` - Schedule configuration
  - Daily CTR at 23:00 for transactions > 300M VND
  - Weekly SAR summary on Mondays at 06:00
  - Monthly compliance report on 1st at 08:00
- `generate_daily_ctr()` - Generates daily CTR from database
- `generate_weekly_sar_summary()` - Aggregates weekly SAR stats
- `generate_sar_from_case()` - Converts AML case to SAR
- `SbvMonthlyComplianceReport` - Monthly combined report

### 4. Updated `crates/ramp-compliance/src/reports/mod.rs`
Added exports for all SBV types.

### 5. Updated `crates/ramp-compliance/Cargo.toml`
Added `chrono-tz` and `ramp-adapter` dependencies.

## Key Features

### SBV XML Export Format
Both CTR and SAR support XML export compliant with SBV standards:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<SBV_CTR_REPORT xmlns="http://sbv.gov.vn/ctr/v1">
  <HEADER>...</HEADER>
  <FILING_INSTITUTION>...</FILING_INSTITUTION>
  <TRANSACTIONS>...</TRANSACTIONS>
</SBV_CTR_REPORT>
```

### Report Trait Implementation
All reports implement the `Report` trait for:
- `title()` - Human-readable title
- `created_at()` - Creation timestamp
- `format_csv_header()` / `format_csv_row()` - CSV export
- `report_type()` - Type identifier

### Scheduler Configuration
```rust
SbvScheduleConfig {
    daily_ctr_enabled: true,
    daily_ctr_time: "23:00",
    weekly_sar_enabled: true,
    weekly_sar_day: 1, // Monday
    monthly_report_enabled: true,
    auto_submit: false, // Keep as Draft until manual review
}
```

## Usage Example
```rust
use ramp_compliance::reports::{
    SbvCtrReport, SbvFilingInstitution, SbvReportScheduler, SbvScheduleConfig,
};

// Generate daily CTR
let scheduler = SbvReportScheduler::new(pool, SbvScheduleConfig::default());
let ctr = scheduler.generate_daily_ctr(tenant_id, Utc::now()).await?;

if let Some(report) = ctr {
    let xml = report.to_sbv_xml();
    // Submit to SBV...
}
```
