# Task T-3.1.5 Handoff: Intent Timeout Handling

## Summary
Implemented timeout handling for intents (Payin, Payout, Trade) including auto-expiration and event notification.

## Changes
1. **New `TimeoutService`** (`crates/ramp-core/src/service/timeout.rs`):
   - Handles checking for expired intents
   - Executes state transitions (EXPIRED, TIMEOUT)
   - Emits status change events

2. **Updated `IntentTimeoutJob`** (`crates/ramp-core/src/jobs/intent_timeout.rs`):
   - Refactored to use `TimeoutService`
   - Runs periodically to process expired intents

3. **Updated Service Logic**:
   - `PayinService`: Set expiration to 30 minutes
   - `PayoutService`: Set expiration to 24 hours
   - `TradeService`: Set expiration to 5 minutes

4. **Updated Repository**:
   - `IntentRepository`: Updated `list_expired` query to exclude additional terminal states

## Verification
- Unit tests added for `TimeoutService`
- Unit tests updated for `IntentTimeoutJob`
- Timeout rules verified in service creation methods
