## Deployment Instructions

1. **Create S3 credentials secret**:
```bash
kubectl create secret generic backup-s3-credentials \
  --namespace rampos \
  --from-literal=ENDPOINT="https://s3.amazonaws.com" \
  --from-literal=BUCKET="rampos-backups" \
  --from-literal=ACCESS_KEY_ID="AKIAXXXXXXX" \
  --from-literal=SECRET_ACCESS_KEY="xxx"
```

2. **Apply backup jobs**:
```bash
kubectl apply -k k8s/base/
```

3. **Verify deployment**:
```bash
kubectl get cronjobs -n rampos
```

## Security Considerations

- All backup jobs run as non-root (UID 1000)
- Seccomp profile: RuntimeDefault
- All capabilities dropped
- S3 credentials stored in Kubernetes secrets (recommend External Secrets Operator)
- Backup files encrypted at rest in S3 (enable server-side encryption)

## Dependencies

- S3-compatible object storage (AWS S3, MinIO, etc.)
- AWS CLI or compatible tool in backup containers
- Network connectivity from cluster to S3 endpoint

## Notes for Operations

- Consider enabling PostgreSQL WAL archiving for true 5-minute RPO
- For production, implement cross-region S3 replication
- Monthly DR tests are critical - schedule in maintenance calendar
- Monitor backup alerts in PagerDuty/Alertmanager
