# T-10.4: Usage-based Billing - Handoff

## Summary
Implemented usage-based billing infrastructure, allowing RampOS to track API calls, transaction volume, and other metrics for enterprise tenants. Includes a middleware for automatic API usage tracking.

## Deliverables

### Core Logic (`crates/ramp-core/src/billing/`)
- `mod.rs`: `BillingService` trait and `InMemoryBillingService` implementation.
- `metering.rs`: `UsageMeter` for tracking and aggregating metrics (API calls, volume, storage, users).
- `stripe.rs`: `StripeClient` adapter (mocked) for subscription and invoice management.

### Database (`migrations/019_usage_billing.sql`)
- `billing_meters`: Definitions of what to track.
- `usage_events`: Raw usage logs.
- `pricing_plans`: Tier definitions (Starter, Growth, Enterprise).
- `invoices`: Generated invoices.

### Middleware (`crates/ramp-api/src/middleware/billing.rs`)
- `usage_metering_middleware`: Asynchronously records 1 API call per authenticated request.
- Integrated into `api_v1` router stack in `router.rs`.

## Configuration
- `BillingConfig` in `mod.rs` controls:
  - Stripe keys.
  - Free tier limits.
  - Sync interval.

## Integration Notes
- `BillingService` added to `AppState`.
- Middleware applied *after* `auth_middleware` (so we have `tenant_id`) but *before* business logic.
- **Performance**: Usage recording is spawned in a separate task (`tokio::spawn`) to avoid blocking the response.

## Testing
- Unit tests in `mod.rs` (checking free tier logic, usage recording).
- Verify that accessing `GET /v1/balance/:user_id` increments the API call counter for the tenant.
